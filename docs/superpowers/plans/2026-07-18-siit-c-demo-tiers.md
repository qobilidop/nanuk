**Date:** 2026-07-18
**Spec:** [SIIT demo design](../specs/2026-07-18-siit-demo-design.md), "Demo tiers".
**Depends on:** Plans A and B (complete on branch `siit`).
**Status:** Ready.

**Goal:** Make the SIIT translator experienceable: the browser playground
runs it on real packets, and a SimBricks scenario pings across address
families through the Verilator'd core.

**Architecture:** Browser tier first (spec ordering): `siit` joins the
playground program selector with v4/v6 presets and the composed PP→MAP run
against baked siit tables; wheels rebuilt (they now carry the Movi IR op and
`s.const`). SimBricks tier second: a two-guest topology where a v4-only host
and a v6-only host converse through `nanuk_switch` running the siit pair —
static neighbor/route entries make the bump-in-the-wire translator work
without ARP/ND gymnastics.

## Global constraints

- Zero ISA / Sail / RTL / vector / asm / twin changes. The scapy boundary
  holds: nothing scapy-touching enters wheels or web/.
- Playground conventions per docs/superpowers/specs/2026-07-11-playground-v1/v2
  design docs and the examples-as-content doctrine: editor seeds are
  standalone example text; the bridge owns any inlined rig with a tripwire
  test holding it identical to the example source.
- Python via `./dev.sh`; web via npm in web/ (host is fine — check
  web/package.json scripts and existing CI); SimBricks runs via the
  documented container flow in benchmarks/e2e (macOS host, retry_make
  lesson applies).
- Deep-link contract: `?program=siit&preset=<name>` must work.
- Casing/copy: "Nanuk", "SIIT translator", the CLAT one-liner where the
  playground describes the program.

## Frozen decisions

- **Playground scope**: program selector entry + presets + composed run +
  provenance panes working for both siit twins. NO new UI panels; the
  existing debugger/result panels must just work. Presets (from the
  committed vectors, NOT regenerated by scapy in web tooling): a v4→v6 UDP
  6052 packet, a v6→v4 UDP packet, a v4 EAMT packet, a v4 ICMP echo, and one
  drop (ttl_expired) — pulled from benchmarks/siit/vectors/*.json by the
  preset generator script (extend web/scripts/gen_presets.py; vectors are
  already committed bytes, so no scapy dependency enters web/).
- **Bridge tables**: siit_tables()'s t0/t1/t2 values for DEMO_SIIT are baked
  into the bridge config (constants, like the existing demo FDB), never
  imported from testkit at runtime.
- **SimBricks scenario** (`benchmarks/e2e/`): two QEMU guests + one
  nanuk_switch running examples/siit/{parse,translate}.asm with DEMO_SIIT
  tables + the t3 flood table (2-port). Addressing: v4 guest 198.51.100.2;
  v6 guest 2001:db8:1::c001 (the EAMT pair's v6 side). v4 guest reaches the
  v6 guest at 192.0.2.1 (EAMT); the v6 guest sees the v4 guest as
  64:ff9b::c633:6402 (6052) and routes 64:ff9b::/96 via its interface with a
  static neigh entry for the v4 guest's MAC; the v4 guest routes
  192.0.2.0/24 with a static ARP entry for the v6 guest's MAC. Beats:
  (1) ping 192.0.2.1 from the v4 guest — ICMP echo translated both ways,
  expect 10/10; (2) iperf UDP then TCP across the translator; (3) negative
  gate: TTL=1 ping (`ping -t 1`) → 100% loss (ttl_expired drop, no ICMP
  error — documented behavior).
- **Runner**: one script `benchmarks/e2e/run_siit.sh` following
  build_component.sh/run_beats12.sh conventions (repo-root assertion lesson:
  the script must assert it resolved the repo root).
