# Stage 3: Nanuk IR — Implementation Plan

**Goal:** extract the eDSL's internal representation into the public protobuf **Nanuk IR** (the hub of the project's satellite architecture) and split the pipeline into eDSL → IR → assembly, with byte-identical assembly output vs. the current direct path.

**Altitude** (per project design): parser-level semantics *above* instruction selection/register allocation — extract/advance/mark/emit-SMD/dispatch over SSA-ish values — not a mirror of the ISA. The IR must be **closed under optimization** (e.g. can express merged/widened extracts) and total (no undefined constructs).

## Schema (`compiler/nanuk_ir/nanuk_ir.proto`, package `nanuk.ir.v0`)

```proto
syntax = "proto3";
package nanuk.ir.v0;

message Program {
  uint32 ir_version = 1;          // = 1
  repeated State states = 2;      // states[0] is the start state
}
message State {
  string name = 1;
  repeated Op ops = 2;
  Terminator terminator = 3;
}
message Op {
  oneof op {
    Extract extract = 1;          // value_id = extracted bits
    Advance advance = 2;
    Mark mark = 3;                // present-bit + offset record
    EmitSmd emit_smd = 4;
    Shift shift = 5;              // value_id = src << amount
  }
}
message Extract { uint32 value_id = 1; uint32 bit_offset = 2; uint32 width = 3; }
message Advance {
  oneof amount { uint32 const_bytes = 1; uint32 value_id = 2; }
}
message Mark { uint32 hdr_id = 1; bool emit_sethdr = 2; }   // false = re-anchor only
message EmitSmd { uint32 value_id = 1; uint32 slot = 2; }
message Shift { uint32 value_id = 1; uint32 src_value_id = 2; uint32 amount = 3; }
message Terminator {
  oneof kind { Dispatch dispatch = 1; Halt halt = 2; Goto goto = 3; }
}
message Dispatch {
  uint32 value_id = 1;
  repeated Case cases = 2;        // ordered; first match wins
  Terminator default = 3;         // Halt or Goto (not nested Dispatch)
}
message Case { uint64 match = 1; string target_state = 2; }
message Halt { bool drop = 1; }
message Goto { string target_state = 1; }
```

Notes: `bit_offset` is relative to the cursor at the point of the op (the frontend has already resolved header anchoring — anchor/epoch bookkeeping is a *frontend* concern, not IR); value ids are unique per program (SSA-ish, no reassignment); widths 1–64.
