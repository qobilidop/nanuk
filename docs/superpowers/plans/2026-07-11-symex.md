# Symbolic Executor Satellite (parser IR, v1)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `nanuk_ir.symex`: Z3-based path enumeration over parser IR Programs — per-path packet constraints, witness packets (path-coverage corpus generation, the design doc's headline payoff), predicted results, and unreachable-state detection — with every witness differentially validated on interp AND the golden emulator.

**Architecture:** DFS over states carrying (Z3 constraint list, concrete step count, symbolic-or-concrete cursor). Packet = Z3 `Array(BitVec16, BitVec8)` + symbolic `plen`; EXT concatenates `Select`s at (possibly symbolic) byte indices with bit alignment done in Z3; every EXT/ADV forks an in-bounds path and a header-violation path when both are satisfiable; Dispatch forks per case (with accumulated ≠ constraints) + default; loops bounded by an `unroll` visit limit per state (the step budget alone allows 256-deep paths — pragmatics first). Step accounting mirrors interp.py exactly, so predicted `steps` is exact per path.

**Tech Stack:** z3-solver (new compiler dev-group dep; never in wheels), existing interp as the oracle chassis.

## Interfaces (frozen)

```python
@dataclass(frozen=True)
class SymPath:
    trace: tuple[str, ...]        # state names visited
    verdict: int                  # predicted (0/1/2)
    error: int                    # predicted error code
    steps: int                    # exact (mirrors interp cost model)
    witness: bytes | None         # model packet (None if unsat — not emitted)
    witness_plen: int

symex(program, *, max_packet=128, unroll=3, max_paths=512) -> list[SymPath]
gen_corpus(program, **kw) -> list[bytes]          # deduped witnesses
reachable_states(program, **kw) -> set[str]
```
