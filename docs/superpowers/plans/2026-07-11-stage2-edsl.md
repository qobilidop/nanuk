# Stage 2: Python eDSL — Implementation Plan

**Goal:** a Python eDSL in `lang/` that expresses parser programs at protocol level (header layouts + parse states) and compiles directly to nanuk assembly text (per the project design: monolithic, no IR — the IR is extracted later as stage 3). The demo program rewritten in the eDSL must be **behaviorally identical** to the hand-written `examples/l2l3l4/parse.asm` over the full pcap corpus (same verdicts, hdr present/offsets, SMD, payload offsets; instruction text/steps may differ).

**Shape of the language** (deliberately surfaces ISA v0's limits — that's the pedagogy):

```python
from nanuk_lang import Header, Parser

eth = Header("eth", dst=48, src=48, ethertype=16)
vlan = Header("vlan", tci=16, ethertype=16)
ipv4 = Header("ipv4", version=4, ihl=4, tos=8, total_len=16, ident=16,
              flags_frag=16, ttl=8, proto=8, csum=16, src=32, dst=32)
udp = Header("udp", sport=16, dport=16, length=16, csum=16)

p = Parser()

@p.state(start=True)
def start(s):
    s.mark(eth, hdr_id=0)                 # SETHDR
    s.smd(s.extract(eth.dst), slot=0)     # EXT + STMD (3 units from width 48)
    ety = s.extract(eth.ethertype)
    s.advance(14)                          # ADVI eth.byte_len
    s.dispatch(ety, {0x8100: vlan_state, 0x0800: ipv4_state},
               default=s.accept)           # MOVI+BEQ chains; HALT accept

@p.state()
def vlan_state(s):
    ...
    s.goto(dispatch_common)                # JMP / fallthrough

# variable advance: only shifts exist in v0 —
    ihl = s.extract(ipv4.ihl)
    s.advance(ihl << 2)                    # SHL + ADVR; ihl*4 sugar allowed

asm_text = p.compile()
```

- `Header(name, **fields)`: ordered fields, bit widths; computes bit offsets and `byte_len`; multi-byte fields must be ≤64 bits; header must be whole bytes.
- Field references resolve to (bit_offset, width) *relative to the cursor at state entry*; `advance` shifts the in-state offset tracking. Extract after advance uses updated relative offsets — the compiler tracks the cursor delta within each state (statically known amounts only; a register advance resets tracking to "unknown", after which extracts must come from a freshly `mark`ed base = compile error otherwise, with a clear message).
- Values are opaque handles; the only operator is `<<`/`*2**k` (v0 has SHL only). Comparisons happen only inside `dispatch` (register-register via materialized MOVI constants ≤16 bits).
- Register allocation: 4 GPRs, per-state linear allocation, "out of registers" is a compile error. `s.accept`/`s.drop` emit HALT.
- Output: assembly text consumed by the existing `nanuk_spec.asm` assembler — encodings stay owned by spec/. `lang/` depends on `spec/python` only for tests (assembling + running the rig).

**Files:** `lang/pyproject.toml` (uv, path deps nanuk-spec for tests), `lang/nanuk_lang/{__init__,header,parser,compile}.py`, `lang/nanuk_lang/programs/l2l3l4.py` (the demo, importable), `lang/tests/test_header.py`, `lang/tests/test_compile.py` (unit: emitted asm patterns per construct, error cases: out-of-registers, extract-after-unknown-advance, >16-bit dispatch constant), `lang/tests/test_parity.py` (the stage-2 done criterion: compile the eDSL demo, run the same 9-case corpus through nanuk-emu via the spec harness, compare full ParseResult per packet against the assembled `examples/l2l3l4/parse.asm`).
