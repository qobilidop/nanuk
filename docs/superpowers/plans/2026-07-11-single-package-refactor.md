# Single-Package Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Collapse the six Python distributions into one `nanuk` package under `python/`, per `docs/superpowers/specs/2026-07-11-single-package-design.md`.

**Architecture:** One mechanical migration commit (git mv + textual import rewrite + one pyproject + plumbing rewire), then a living-docs commit. The existing test suite is the safety net — no new tests; the gate is the full suite staying green plus the Pyodide wheel path.

**Tech Stack:** uv (single project), hatchling, pytest (prepend import mode), Pyodide/micropip on the web side.

## Global Constraints

- `nanuk/__init__.py` is docstring-only; only `nanuk/rtl/*` may import amaranth.
- `nanuk_ir.proto` and `nanuk_ir_pb2.py` keep their filenames; the pb2 file's contents are never edited.
- Historical records (`guide/notes/`, `docs/superpowers/plans|specs/` except this plan's checkboxes) are not rewritten.
- Required dep: `protobuf>=7.35` only. Extra `rtl`: `amaranth[builtin-yosys]>=0.5`. Everything else is the `dev` group.
- Tree must be green (full suite + ruff) before the migration commit.
